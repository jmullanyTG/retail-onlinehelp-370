Site Modeling

Site modeling allows users to select sites for supersession functionality, similar to inventory supersessions. It may also be used to create modeling for new sites based on product levels. Item (inventory level) supersessions and site model quantities will be combined when calculating history for the purposes of demand forecasting.

Site Model Options

Site model options functions as the “header” for the supersession/model. Users may select the start and end dates of the model, as well as its active status.

  • Supersession Start Date – The start date for the model. This is the first period of historical sales which will be considered.
  • Supersession End Date – The end date for the model. This is the final period of historical sales which will be considered. If this date is beyond the date on which the new store will have accumulated actualized sales, the model will continue to be used to represent sales history.
  • Active – A flag used to determine the active status of the supersession.
  • Site Opening Date – The date on which a site is set to open. To be used for new store openings. This is not a user editable field. 
  • Created Date – Date of creation for the supersession.
  • Created By – User credited with the creation.
  • Last Updated – Date of the most recent update to the supersession.
  • Last Update By – User credited with the most recent update.
  • Model Last Process – The supersession model will need to be reprocessed periodically. This task should be done as part of a job which will coincide with sales history updates (i.e. nightly job).

Site Model Details

Site model details will allow users to determine the product levels which will contribute to the creation of the model and their relative contributions.

  • Product Level – The level within the product hierarchy which may be modeled. The available hierarchy levels are controlled through a system policy setting.
  • Model Site – The site which will be used to create the model.
  • Model Site Opening Date – The opening date for the modeled site. This is not a user editable field.
  • Factor – The ratio which will be applied to the sales history of the modeled site. A value of “1” will apply 100% of the sales to the model.
  • Active - A flag used to determine the active status of the product level within the model.
  • Created Date – Date of creation for the model line.
  • Created By – User credited with the creation.
  • Last Updated – Date of the most recent update to the model line.
  • Last Update By – User credited with the most recent update.
