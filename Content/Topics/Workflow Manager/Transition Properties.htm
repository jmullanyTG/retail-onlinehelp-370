Transition Properties

Property

Description

Name

The name of the transition. Default value is [Start Status Name] -> [End Status Name].

Start Status

The name of the status that the transition starts from.

End Status

The name of the status that the transition ends at.

Weight

When a start status has a trigger method that automatically advances the workflow to the next status, this value determines which status is transitioned to in case there are more than possible "next" statuses.

For example, the start status "Review" may have two possible next statuses: "Approved" and "Changes Needed". Validation criteria in the "Review" status determines which status is transitioned to, based on the weight values of the "Approved" and "Changed" statuses.
Weight values are required for Order Header workflows and optional for all others.
Transition Access The roles or users permitted to move the worfklow from the start status to the end status. Click Edit to view/change this list.
