Breaking Changes

Allocation

Updates to the curve mapping process (usp_Curve_Common_MapCurveFactorsToDateSeries) will necessitate examination of any custom code and any pre-existing curves.

Database

Introduction of a cleanup process for the “temp” schema will necessitate examination of any custom code which references objects within that schema.

Engine

Multiple changes to the general processing architecture within JustEnough will necessitate review of all engine/system mapping related system policy values. Please consult the updated architecture documentation and installation/upgrade guides for additional exposition.

Event Manager

If upgrading from v3.6.2.6 or earlier, the Event Manager module will automatically migrate event slot sets, templates and display format configurations to be compliant with the new Event Slot Set Template control. The number of conversions will determine the amount of time to complete. DO NOT close the application or interrupt this process.

Financial Planning

Adjustments made to plan initialization procedures related to historical inventory valuation will necessitate any production instances which have inventory archive imports set up to examine the shape of data within the historical inventory import.

  • If the import is set to match the data specification (including Total Cost / Total Revenue in the import) no change is required in import structure, but adjustments must be made to the data to divide by Units so that data is stored as Per Unit cost.
  • If the import is set to take in Per Unit values, adjustments must be made to the import to specify that the values coming in are already at the unit level with the new parameter.

Due to a change related to disallowing negative values in snapshot start and end dates, snapshot logic may be altered in an unexpected way if any of those values needed to be negative in any custom implementations.

Order Manager

With the introduction of the logistics instruction field, any custom code which references or calls OrderManager.Order_OrderHeader_AddUpdate will need to be reviewed.
