Allocation

The following enhancements are now available in the Allocation module.

Enhanced size curve validation

You can set up pass/fail tests to ensure each of your sites had enough qualified data left over after data cleansing to calculate a valid size curve. These tests are based on user-defined thresholds that measure variants, sales units, and sizes at a site within the item filter selected for the size curve.

Validations are conducted on the current size curve calculated for each site (the validation process itself does not recalculate curves). Results for each site are displayed in the Size Curves grid.

For more information, see Validating Enhanced Size Curves in the Allocations online help.

Option to skip retrend calculations for strategy lines that do not use retrend forecasts

To improve performance, you can now skip running retrend calculations for strategy lines that doe not use retrend forecasts. This requires two steps:

  1. Change the setting of the new Always Run Retrend Calculation system policy to False. (It is set to True by default).
  2. In the Allocation Strategy screen, make sure that the Use Re-Trend column is "unchecked" for each strategy line that is not using retrend forecasts.

Note: Retrend forecasts will always be calculated for strategy lines whose Use Re-Trend column is checked/selected.

Aged Inventory priority option for product groups now ranks by last receipt date

In the Product Groups screen, the Group Allocation priority option, Aged Inventory, used to rank products by the date they were first stocked (FirstStockedDate in tbl_Inventory). In order to better identify truly stagnant items, Aged Inventory now ranks products by their last receipt date (LastReceiptDate in tblInventory).

Note: If a LastReceiptDate value is NULL, JustEnough falls back to using FirstStockedDate.

New Re-Trend Method column in Allocation Strategy grid

A new column in the Allocation Strategy grid lets you choose the method for obtaining a strategy line's retrend forecast.

Column Name

Description
Re-Trend Method

The type of retrend forecast used for this strategy line. Options are:

  • Default - Uses the option set in the Default Forecast Retrend Method To Use system policy.
  • PID - Use the standard retrend calculation to create a retrend forecast.
  • Monte Carlo - Use the Monte Carlo retrend method to create a retrend forecast
  • Demand Forecast - Use a generated inventory-level demand forecast as the retrend forecast. Requires the Demand Forecasting module and an existing demand forecast. (Demand forecasts are not generated as part of Allocation execution.)

For example, customers with both the Allocation and Demand Forecasting modules can now get moving-average based, season-affected demand signals for allocations by:

  1. Using shape forecasts to generate demand forecasts.
  2. Setting the Re-Trend Method for the allocation strategy to Demand Forecast.
