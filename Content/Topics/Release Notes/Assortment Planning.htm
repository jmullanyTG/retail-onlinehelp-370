Assortment Planning

The following enhancements are now available in the Assortment Planning module.

Additional Filter Metadata

The range of metadata you can use to create product filters in Assortment Planning has been extended to include additional assortment plan metrics. You can use this new metadata to create exception filters (for example, placeholders that are missing size curves) or identify placeholders based on criteria beyond strict product levels (for example, placeholders with phase-in dates in the next 30 days).

The new metadata fields are listed by group below.

For descriptions of these new metadata fields, see the descriptions of their associated metrics in the following sections of the Assortment Planning help:

  • Main Planning View
  • Cluster Set View

New Group: Assortment Plan Data - Cluster

This newly added group of metadata fields lets you filter plan metrics at the cluster level. When you filter by these fields, JustEnough compares the filter criteria to each assorted cluster individually. If any cluster meets the filter criteria, the placeholder is included in the filter. For example, filtering by Cluster- Total Sales (Rev) compares the filter criteria against the total sales revenue of each individual cluster the placeholder is assorted to. If any cluster meets this criteria, the placeholder is included in the filter.

New Group: Assortment Plan Data - Placeholder

This newly added group of metadata fields lets you filter metrics at the placeholder level. When you filter by these fields, JustEnough compares the filter criteria to the total placheolder. For example, filtering by Placeholder - Total Sales (Rev) compares the filter criteria against the total combined sales revenue of all clusters a placeholder is assorted to.
